# 考种平台 - 使用说明

## 📋 项目简介

这是一个基于 **YOLOv8** 和 **PyQt5** 开发的现代化考种平台，支持大豆、小麦等多种作物的自动考种。

### 主要功能

✨ **考种分析** - AI自动检测、计数、测量、称重  
📷 **图像采集** - 高效的相机图像采集工具  
⚙️ **系统设置** - 可视化ROI框选、参数配置  

---

## 🚀 快速开始

### 1. 环境安装

```bash
# 使用 conda 创建环境
conda env create -f environment.yaml
conda activate SeedTest

# 或使用 pip 安装
pip install -r requirements.txt
```

### 2. 准备模型文件

将 `.pt` 模型文件放入 `models/` 文件夹，并编辑 `models/model_mapping.csv`：

```csv
soybean_obb_1024.pt,大豆-旋转框-1024
wheat_det_640.pt,小麦-普通检测-640
```

### 3. 启动程序

```bash
python start.py
```

## 📖 使用教程

### 一、考种流程

#### 1. 选择模型
- 点击主页的"考种"卡片
- 在弹出的对话框中选择合适的模型
- 点击"确定"进入考种页面

#### 2. 连接设备
**相机：**
- 点击"扫描相机" → 选择相机 → "打开相机"
- 调整曝光、增益、帧率等参数
- 点击"应用参数"

**天平（可选）：**
- 点击"扫描天平" → 选择串口 → "打开天平"
- 状态显示"已连接"且实时显示重量
- 如需清零，点击"清零"按钮

#### 3. 拍照与处理
1. 点击"开始预览" - 实时查看相机画面
2. 点击"处理图像" - AI自动检测分析
3. 调整"置信度"和"面积筛选"滑块优化结果
4. 确认结果无误后，点击"保存数据"

#### 4. 数据管理
- 切换到"数据管理"标签页
- 查看历史记录、导出Excel、查看图像
- 支持删除单条记录或清空所有数据

---

### 二、图像采集

纯图像采集功能，不做AI处理，适合快速连拍建立数据集。

1. 扫描并打开相机
2. 设置保存路径和文件前缀
3. 点击"开始预览"
4. 点击"拍照"进行连续拍摄

---

### 三、系统设置

#### 1. 相机与ROI设置

**ROI可视化框选：**
1. 扫描并打开相机
2. 点击"开始预览" - 显示全分辨率图像
3. 用鼠标在预览区域**拖拽绘制矩形**
4. 松开鼠标后，ROI参数自动更新
5. 点击"应用ROI"确认
6. 最后点击底部"保存设置"

**相机参数：**
- 曝光时间、增益、帧率
- cm/像素 - 用于将像素尺寸转换为实际尺寸

#### 2. 天平设置
- 串口号（如 COM3）
- 波特率（默认 9600）
- 超时时间

#### 3. 检测设置
- 默认置信度 - 滑块初始值
- 默认面积筛选 - 滑块初始值
- 推理置信度 - 模型推理时的阈值

#### 4. 路径设置
- 模型文件夹
- 图像保存路径
- 数据文件路径

⚠️ **注意：** 修改路径配置需重启程序生效！

## 🔧 常见问题

### Q1: 未检测到相机？
**解决方案：**
- 检查相机USB连接
- 确认已安装海康威视SDK
- 重新扫描相机

### Q2: 天平数据异常（负数/跳变）？
**解决方案：**
- 检查串口连接
- 点击"清零"按钮
- 重新插拔天平USB线，然后重新扫描

### Q3: 模型推理很慢？
**解决方案：**
- 检查是否使用了GPU（需安装CUDA版本的PyTorch）
- 使用更小的模型（如 640 分辨率）
- 减小图像分辨率或ROI区域

### Q4: 检测结果不准确？
**解决方案：**
- 调整置信度阈值（降低可检测更多目标）
- 调整面积筛选（过滤太小的误检）
- 优化相机参数（曝光、增益）
- 使用更适合的模型

### Q5: 保存的图像在哪里？
**答案：**
- 原始图像：`data/images/`
- 处理后图像：`data/processed/`
- 数据记录：`data/records.json`

## 🎨 模型类型说明

### OBB (Oriented Bounding Box)
- **特点：** 旋转框检测，适合密集排列的目标
- **优势：** 长宽测量更准确
- **文件命名：** `*_obb_*.pt`

### DET (Detection)
- **特点：** 普通矩形框检测
- **优势：** 推理速度快
- **文件命名：** `*_det_*.pt` 或其他

### SEG (Segmentation)
- **特点：** 像素级分割（未来支持）
- **优势：** 最精确的边界
- **状态：** 🚧 开发中

## ⚡ 性能优化建议

1. **使用GPU加速**
   ```bash
   # 安装CUDA版PyTorch
   pip install torch torchvision --index-url https://download.pytorch.org/whl/cu118
   ```

2. **优化图像分辨率**
   - 在设置中调整ROI，只保留有效区域
   - 减小不必要的背景

3. **选择合适的模型**
   - 640分辨率模型：速度快
   - 1024分辨率模型：精度高
